# Boilerplate v1.0.1

Release axis: boilerplate

Catalog contracts: core-v1 and index-v2 (unchanged)

Profiles: bootstrap, standard (default and legacy), and scaled (unchanged)

Compatible soku: soku/v0.1.1 for core lifecycle; soku/v0.1.2 recommended

Migration from previous release: v1.0.0 to v1.0.1 (six catalog-managed sources changed; no removals, ownership transitions, or mergeable-path changes)

Provider compatibility: provider-v1 (unchanged); soku/v0.1.2 fully supports optional deprecated legacy ref

Companion tag: none

This PATCH release publishes the reviewed template and verification fixes that
cannot be delivered to immutable `v1.0.0`. It retains `core-v1`, `index-v2`,
manifest-v1, provider-v1, the `bootstrap`, `standard`, and `scaled` profile
composition, core/provider ownership, and the `.gitignore` and `.editorconfig`
mergeable-path contract.

The migration from `v1.0.0` changes exactly these six catalog-managed sources:

- `templates/_shared/ci/downstream-ci.yml`
- `templates/java-spring/pom.xml`
- `templates/javascript-typescript-node/package.json`
- `templates/javascript-typescript-node/package-lock.json`
- `templates/python/pyproject.toml`
- `templates/python/requirements-lock.txt`

The shared downstream workflow now renders JavaScript output that passes the
current formatter. The Java template imports the reviewed Jackson BOM. The
JavaScript dependency snapshot overrides vulnerable transitive `tmp` versions,
and the Python template and lockfile retain the reviewed formatter command and
dependency snapshot corrections.

There are no file removals, ownership transitions, mergeable-path changes, or
manual actions for a clean `v1.0.0` snapshot. If a downstream project modified
an affected core-managed file, `diff` and `upgrade` apply the existing lifecycle
drift and conflict rules; the release does not overwrite that customization
silently.

`soku/v0.1.1` remains compatible with the core lifecycle. Published
`soku/v0.1.2` is the recommended client and fully supports provider-v1 bundles
that omit deprecated legacy `ref` or retain any well-formed legacy value. No CLI
contract, manifest schema, catalog schema, profile, provider major version,
recovery behavior, or exit code changes in this boilerplate release.

Immutable `v1.0.0` retains its known limitations: generated JavaScript fails
`init --verify`, and its dependency snapshot predates the current `tmp` and
Jackson fixes. Those historical objects are not moved, deleted, or reused.

Release validation requires Markdown/YAML/action semantics, release-note and
tag regression, sync parity, full Go unit/race/vet/format/import/lint, lifecycle
and five-target package gates, every runtime-template and hosted database/cloud
gate, dependency/vulnerability, secret, and license checks, and the aggregate
`Validation Gate`. Public lifecycle smoke must cover a fresh four-application-
stack `init --yes --verify`, clean `status`, same-release `diff`, no-op
`upgrade`, and a separate `v1.0.0` to `v1.0.1` diff and upgrade.

Release preflight uses `boilerplate-tag` set to `v1.0.1` and `cli-tag` set to
`soku/v0.1.2`; its publish job must be skipped. Delivery requires a separately
approved signed annotated `v1.0.1` tag on the reviewed `main` commit. The
boilerplate GitHub Release receives no CLI archives or checksum asset.
