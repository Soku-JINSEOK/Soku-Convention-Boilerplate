# Boilerplate v1.0.5

Release axis: boilerplate

Catalog contracts: core-v1 and index-v2 (unchanged)

Manifest and provider contracts: manifest-v1 and provider-v1 (unchanged)

Profiles: bootstrap, standard (default and legacy), and scaled (unchanged)

Compatible soku: soku/v0.1.4

Migration from previous release: v1.0.4 to v1.0.5 changes only the JavaScript
and TypeScript `.prettierignore`; no removals, ownership transitions, or
mergeable-path changes

Provider compatibility: provider-v1 unchanged; optional deprecated legacy ref
behavior is unchanged

Companion tag: none

This corrective PATCH release excludes the lifecycle-owned `.soku/` directory
from JavaScript and TypeScript formatting checks. Public migration smoke for
immutable `v1.0.4` found that `npm run format:check` inspected
`.soku/manifest.json` after a successful upgrade. Fresh initialization did not
expose the defect because verification precedes the final manifest write.

The migration from `v1.0.4` changes exactly one catalog-managed source:

- `templates/javascript-typescript-node/.prettierignore`

There are no removals, ownership transitions, mergeable-path changes, or
manual actions for a clean `v1.0.4` snapshot. Existing public tags and Releases
remain immutable. Published `soku/v0.1.4` can initialize and migrate to this
single-axis boilerplate correction without a new CLI release.

Verification requires catalog rendering regression coverage, the full
repository and runtime-template validation, three-OS lifecycle, five-target
package snapshot, dependency and vulnerability audits, sync parity, and the
aggregate `Validation Gate`. Public smoke must cover fresh four-application-
stack initialization, post-manifest formatting, status, same-release diff,
no-op upgrade, and migration from `v1.0.4` to `v1.0.5`.

Release preflight uses `boilerplate-tag` set to `v1.0.5` and `cli-tag` set to
`soku/v0.1.4`; its publish job must be skipped. Delivery requires separately
approved signed annotated `v1.0.5` on the reviewed source commit. The
boilerplate GitHub Release receives no CLI archives or checksum asset.
